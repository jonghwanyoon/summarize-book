# 기술/IT 도서 평가 가이드

## 대상 독자 프로필
- **주 독자층**: 현직 개발자, IT 엔지니어, CS 전공 학생, 기술 전환 희망자
- **기대 요소**: 실무에 바로 적용할 수 있는 코드와 아키텍처, 기술의 원리에 대한 깊은 이해, 최신 트렌드와 모범 사례
- **독자 특성**: 공식 문서와 온라인 자료에 익숙하며, 책에서 "문서 이상의 가치"를 기대함. 코드 품질과 정확성에 민감

## 요약 스타일
- **톤**: 기술적이고 구체적. 모호한 표현 지양
- **초점**: 핵심 기술 개념, 코드/아키텍처 패턴, 실무 적용 시나리오
- **용어 수준**: 기술 용어 그대로 사용 (번역서의 경우 원어 병기)
- **포함 요소**: 다루는 기술 스택과 버전, 난이도 수준 (입문/중급/고급), 코드 예시의 질적 평가

## 품질 평가 기준
1. **코드 예시의 유무와 품질**: 실행 가능한 코드가 있는가? 복사-붙여넣기해도 돌아가는 수준인가? 에러 처리와 엣지 케이스를 다루는가?
2. **기술 트렌드 시의성**: 출간일 기준으로 다루는 기술이 현역인가? 이미 deprecated되거나 대체된 기술이 아닌가?
3. **실무 적용 가능성**: 개념 설명에 그치지 않고 실제 프로젝트에 적용하는 방법을 보여주는가? 프로덕션 환경의 고려사항을 다루는가?
4. **저자의 실무 경력과 전문성**: 저자가 해당 기술을 실무에서 사용한 경험이 있는가? 오픈소스 기여, 기술 블로그, 컨퍼런스 발표 등의 이력이 있는가?
5. **공식 문서 대비 부가가치**: 공식 문서를 정리한 수준인가, 아니면 경험에서 우러나온 통찰이 있는가? "왜" 그렇게 하는지를 설명하는가?
6. **난이도와 구성의 적절성**: 대상 독자 수준에 맞는 난이도 곡선을 갖추었는가? 기초부터 고급까지 논리적으로 구성되어 있는가?
7. **테스트와 디버깅**: 테스트 코드를 제공하는가? 흔한 에러와 디버깅 방법을 다루는가?

## 헛소리 감지 규칙
1. **"이 기술 하나면 취업/이직 보장"**: 특정 기술 하나로 커리어 성공을 보장하는 주장. 기술 시장은 다면적이며 단일 기술로 해결되지 않음
2. **이미 deprecated된 기술을 최신인 듯 소개**: 출간 시점에 이미 레거시가 된 기술을 미래 기술처럼 포장. 버전 정보와 출간일 교차 확인 필요
3. **공식 문서 복사-붙여넣기**: API 레퍼런스를 그대로 옮겨놓고 "완벽 가이드"로 포장. 저자만의 해석이나 경험이 없음
4. **"풀스택 N일/N주 완성"**: 비현실적 학습 기간 제시. 복잡한 기술 스택을 짧은 기간에 마스터할 수 있다는 환상 조장
5. **깊이 없는 기술 나열 (기술 뷔페)**: 10가지 프레임워크를 각각 2페이지씩 다루며 "올인원" 주장. 어느 하나도 제대로 설명하지 않음
6. **비현실적 성능 주장**: 벤치마크나 근거 없이 "10배 빠름", "무한 확장 가능" 등의 과장
7. **보안/에러 처리 무시**: Happy path만 다루고 보안 취약점, 에러 처리, 예외 상황을 완전히 무시

## 추천 등급 기준
- **★★★★★ 강력 추천**: 실행 가능한 고품질 코드 예시가 풍부하고, 저자의 실무 경험에서 나온 깊은 통찰이 있으며, 공식 문서 이상의 명확한 부가가치가 있는 책. 출간 후 2년 이내 기술이 현역이고, 아키텍처 결정의 "왜"를 설명
- **★★★★☆ 추천**: 코드 예시가 있고 대체로 정확하며, 실무 적용 시나리오를 다룸. 일부 영역에서 깊이가 아쉽거나 특정 버전에 종속적이지만 핵심 개념 학습에 충분한 가치
- **★★★☆☆ 보통**: 개념 설명은 괜찮으나 코드 예시가 부족하거나 품질이 들쭉날쭉. 공식 문서와 차별점이 크지 않으며, 특정 목적(입문용 등)으로만 제한적 가치
- **★★☆☆☆ 비추천**: 코드에 오류가 있거나 deprecated된 기술을 다루며, 공식 문서 수준의 정보만 제공. 실무 적용이 어렵고 저자의 전문성이 의심됨
- **★☆☆☆☆ 경고**: 기술적으로 부정확한 내용이 다수, 실행 불가능한 코드, 이미 사장된 기술을 최신으로 포장, 또는 "이 기술이면 취업 보장" 식의 과대 마케팅 도서
